## Overview

### The challenge

Users should be able to:

- Complete the form and see a success toast message upon successful submission
- Receive form validation messages if:
  - A required field has been missed
  - The email address is not formatted correctly
- Complete the form only using their keyboard
- Have inputs, error messages, and the success message announced on their screen reader
- View the optimal layout for the interface depending on their device's screen size
- See hover and focus states for all interactive elements on the page

### What I learned

In this challenge, I focused on building a form structure with JavaScript while staying aligned with web accessibility principles.

Main things I learned:

- How to validate form fields with JavaScript and show error states
- How focus events can improve user feedback on form inputs
- How to style and manage radio button selection states using JavaScript
- How and why to use the `required` attribute in forms

I did not document every focus warning behavior in detail in this README, but working on focus interactions helped me better understand accessible form UX.
